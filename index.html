Introduction aux Model Context Protocols (MCP) et au Vibecoding

1. Introduction

Dans le cadre de notre démarche vers une automatisation plus efficace du développement logiciel chez Concree, j'ai exploré et testé les Model Context Protocols (MCP). Cette présentation vise à vous introduire à cette nouvelle approche et à discuter de son impact potentiel sur notre manière de coder. Nous aborderons également le concept du "Vibecoding", une vision du futur du développement logiciel.

2. Qu'est-ce qu'un MCP ?

Les MCP sont des protocoles permettant d'automatiser et d'intégrer des contextes de travail dans nos outils de développement. Ils agissent comme des connecteurs intelligents qui facilitent l'interaction entre différents services et applications.

2.1. Expérience avec les MCP

J'ai réussi à intégrer et tester deux MCP dans mon éditeur Cursor :

  • MCP Figma (https://github.com/GLips/Figma-Context-MCP) Integration du design de la fonctionnalite "Evenements" de Lezgo.
  • MCP GitHub (https://smithery.ai) : Utilisation pour interagir avec mon compte GitHub pour push ce projet depuis Cursor.

Les deux intégrations fonctionnent parfaitement, permettant d'automatiser des tâches et de fluidifier mon workflow de développement.

3. Pourquoi adopter les MCP ?

Les MCP offrent plusieurs avantages :

  • Automatisation des interactions entre outils sans besoin d'intervention manuelle.
  • Gain de temps en éliminant les tâches répétitives et fastidieuses.
  • Intégration fluide avec des services tiers comme Figma et GitHub.
  • Amélioration du workflow en centralisant les actions dans un seul environnement.

4. Le Vibecoding : L'avenir du développement ?

Andrej Karpathy explique que le développement d'applications web en 2025 ressemble à l'assemblage de meubles IKEA : un ensemble de services à configurer (écosystème frontend/backend, hosting, base de données, CI/CD, etc.). Cette complexité ne repose pas uniquement sur l'écriture de code mais plutôt sur la configuration et l'orchestration des différents services.

Le Vibecoding décrit une approche dans laquelle les développeurs s'appuient davantage sur l'IA et les MCP pour automatiser ces processus. L'objectif est de permettre à chacun de se concentrer sur la création plutôt que sur la configuration.
